This project is an incredibly alpha-quality experiment in code editor UIs. Its hacked together, so don't expect it to be bug free. Its also changing rapidly. Source code at https://github.com/Sheyne/paren-ui.

How to use: navigate with the mouse or arrow keys (up and down move between arguments). "Enter" enters edit mode. "Esc" exits edit mode. Type to rename things. Vertical elements in parens are function arguments. ctrl-alt-b toggles "horizontal" mode for a cell (I'm in the market for a better shortcut). Note that if you are immediatly inside of a horizontal cell, it unfolds to vertical. Comma makes a new element immeditately after the current one, as does "Enter" if you're already in edit mode. Edits output continuously, be careful of putting the code in an infinite loop, which will currently hang the GUI. Syntax should feel very lisp-like, but (a b c) is written a(b c). Also letrec binds values to identifiers letrec(name1(binding1), ..., nameN(bindingN), valueToExecute).

The default example above computes the summation from for i = low to i = high of func(i).
